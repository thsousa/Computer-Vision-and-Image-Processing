Exercício Xadrez¶

Um grupo de enxadresistas possui um conjunto de partidas armazenados como imagens e deseja criar um sistema para extrair as jogadas de cada partida para anlise. Esse grupo deseja saber o movimento feito em cada rodada, ou seja, a pea movida da casa $x$ para a casa $y$. O grupo j tem programado a estrutura que deseja utilizar e que j foi colocado no arquivo "chess/chess.py". Este arquivo contm as seguintes classes:

  • ChessSquare: Classe que representa uma casa do tabuleiro do Xadrez.
  • Atributos
    • piece_name: String de identificao da pea.
    • x: ndice horizontal da pea no tabuleiro (0 ... 7)
    • y: ndice vertical da pea no tabuleiro (0 ... 7)
  • Move: Classe que representa uma jogada
  • Atributos
    • image_filepath: caminho do arquivo de imagem da jogada.
    • piece_start: ChessSquare da pea antes da jogada.
    • piece_stop: ChessSquare da pea depois da jogada.
  • Chessboard: Classe que representa o tabuleiro de xadrez.
  • Atributos
    • board: vetor de ChessSquare, que armazema todas as casas do tabuleiro de xadrez.
    • template: matriz que representa o tabuleiro em seu estado inicial.
  • Mtodos
    • _init_board: Mtodo que inicializa o tabuleiro com as posies iniciais de cada pea.
    • set_piece: recebe uma string (piece_name) e coloca essa string na posio ("x", "y") do tauleiro. Onde piece_name qualquer string de identificao de pea vlido ou a string vazia "" (representa casa vazia).
    • print_board: Mtodo que imprime o tabuleiro no console com string de identificao para cada casa. (Uso para debug).
    • get_flat_board: Mtodo que retorna o tabuleiro como uma lista de ChessSquare, utilizando uma varredura de cima para baixo, da esquerda para direita.
  • Game: Classe que representa uma partida de Xadrez
  • Atributos
    • moves: Lista de jogadas (objetos tipo Move) utilizado para armazenar as jogadas onde moves[i] representa a jogada i, moves[0] a primeira jogada e moves[len(moves) - 1] a ltima jogada.
  • Mtodos
    • add_move: Adiciona uma jogada na posio len(moves) e aumenta o tamanho de moves em 1.

pedido que seja implementado a funo analyze_chess_game localizada no arquivo "chess/analyze.py", que recebe um diretorio (no caso "images/chess") contendo as imagens das jogadas nmeradas de $1$ a $n$, todas do tipo ".png" (i.e. "images/chess/5.png" corresponde a imagem da jogada 5) e retorne um objeto Game com todas as jogadas armazenadas em seu atributo $moves$.

As imagens nomeadas com nmeros mpares so jogadas de pea branca e nmeros pares de pea preta e a imagem "0.png" a posio inicial do tabuleiro (sempre ser a mesma imagem), cada casa possui dimenses 60x60 pixels, as casas escuras tm intensidade $80$ e as casas claras, intensidade $180$. As peas pretas tm intensidade $0$ e as peas brancas $255$. Os identificadores das peas so definidos abaixo:

  • P: Pawn (Peo)
  • N: Knight (Cavalo)
  • B: Bishop (Bispo)
  • Q: Queen (Rainha)
  • K: King (Rei)

Dentro da classe ChessSquare, no atributo piece_name, a string de identificao de peas recebe a abreviao descrita acima, mais uma letra "W"(White) para peas brancas e "B"(black) para peas pretas, alm disso, mais um nmero de identificao, caso exista mais de uma mesma pea (e.g., "NB1" o cavalo preto nmero 1, note que existem 2 cavalos pretos; "PW5" o peo branco nmero 5, note que existem 8 peos de cada cor). Alm disso, a string de identificao de peas pode ser vazia para indicar que a casa do tabuleiro est vazia.

A cada nova imagem, seu algoritmo deve registrar em um objeto Move, no atributo piece_start, um objeto ChessSquare, com a seguintes informaes:

  • string de identificao da pea movida;
  • a posio inicial da pea movida (posio da pea movida na jogada anterior)

e no atributo piece_stop, um objeto ChessSquare, com a seguintes informaes:

  • a string vazia, se a pea no capturou nenhuma pea adversria, ou identificao da pea capturada, caso contrrio;
  • a posio da pea depois da jogada conforme imagem sendo analisada.

Alm disso, no atributo image_filepath, o caminho do arquivo da imagem sendo analisada. Finalmente, o objeto Move deve ser armazenado no objeto Game.

from IPython.display import Image, display
from chess.analyse import analyze_chess_game

game = analyze_chess_game("images/chess")


for move in game.moves:
        pstart = move.piece_start
        pstop = move.piece_stop
        text = pstart.piece_name + " moves from " + str((pstart.x, pstart.y))
        text += " to " + str((pstop.x, pstop.y))

        if pstop.piece_name is not "":
            text += " and captures " + pstop.piece_name
        text += "\n"    
        display(Image(filename=move.image_filepath))
        print(text)
